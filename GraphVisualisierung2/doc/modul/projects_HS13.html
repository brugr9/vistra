CLJ1-HS13-1

Titel: Travel planner

Beschreibung: The aim of the project is, in a first phase, to allow the user to graphically specify a whole road net as well as a fleet of vehicles, and then, in a second phase, to ask the system in which “best” ways the vehicles can drive from specified start points to specified end points according to various individual criteria for each vehicle: time, consumption, etc…
A road net consists in road stretches with various speed limitations, slopes, impediments, gas stations, etc… A vehicle is likewise characterized by its consumption under variable conditions, tank capacity, speed limit, etc… The characterizations of both roads and cars are part of the project.

Gruppengrösse: 2-3

Anzuwendende Technologien: UML, any of the following programming languages: C/C++/C#/Java, graphs, modeling, optimization.

CLJ1-HS13-3

Titel: Search robot

Beschreibung: The aim of the project is, in a first phase, to allow a user to specify a landscape with various impediments and various ways between those impediments. Then, in a second phase, the user can ask a (software) robot, equipped with distance sensors, to find its path between those impediments from any starting point to any goal point without a priori knowing the landscape. The robot has a distance sensing capability limited to [ - 90 ° , + 90 ° ] of its current moving direction. Besides, it can memorize already run paths.

The characterization of the landscape with the impediments is part of the project.

Gruppengrösse: 2-3

Anzuwendende Technologien: UML, any of the following programming languages: C/C++/C#/Java, modeling.

ERJ1-HS13-6

Titel: Modellierung und Simulation des Schiffsverkehrs durch den Nord-Ostsee-Kanal

Beschreibung: In dieser Arbeit soll eine Softwareumgebung zur Simulation des Schiffsverkehrs auf dem Nord-Ostsee-Kanal erstellt werden. Der Nord-Ostsee-Kanal gehört zu dem am meisten befahrenen Schifffahrtswegen der Welt. Täglich durchfahren Schiffe diese Wasserstrasse. Probleme ergeben sich daraus, dass der grösste Teil der Wasserstrasse nur in eine Richtung befahrbar ist und nur an einigen wenigen Stellen Schiffe in entgegengesetzter Richtung vorbei fahren können. Daraus resultierend ein Optimierungsproblem, nämlich eine Vorrangreihenfolge zu finden, die den Durchsatz maximiert. Bei dieser Projektaufgabe kann der Fokus einerseits auf der grafischen Modellierung und Simulation des Schiffsverkehrs liegen, andererseits aber auf der algorithmischen Lösung des Optimierungsproblems.

Gruppengrösse: 2-3

Anzuwendende Technologien: JAVA

Link: http://www.math.tu-berlin.de/coga/projects/canal/

Betreuer: Jürgen Eckerle

ERJ1-HS13-7

Titel: Intelligentes Fahrermodell für die TORCS-Umgebung

Beschreibung: In diesem Projekt soll ein Fahrermodell entwickelt werden, das ein Fahrzeug in einer virtuellen Umgebung steuern kann. Um genügend Zeit für die Entwicklung eines Fahrermodells zu haben, könnte die TORCS-Umgebung verwendet werden, die eine Plattform für Rennspiele bereitstellt. Hier gilt es einen verfügbaren trivialen Bot so zu erweitern, dass dieser ein Fahrzeug durch eine bekannte oder unbekannte Rennstrecke kollisionsfrei steuern kann. Über eine einfache klar definierte Schnittstelle können die Umgebungsdaten abgefragt und das Fahrzeug gesteuert werden. Falls für ein Einzelfahrzeug hinreichend gut gesteuert werden kann, kann anschliessend die Konkurrenzsituation mit einem zweiten Fahrzeug gemeistert werden. Die Steuerung könnte beispielsweise auf der Basis von Fuzzy-Logik oder Regelbasierten System implementiert werden. Alternativ zur TORCS-Umgebung könnte man eine eigene Umgebung erstellen, die beispielsweise eine bestimmte zu bewältigende Verkehrssituation repräsentiert.

Gruppengrösse: 2-3

Anzuwendende Technologien: JAVA

Links: http://de.wikipedia.org/wiki/TORCS

Betreuer: Jürgen Eckerle

SWP1-HS13-11

Titel: Hochregallager Simulationssoftware

Beschreibung: Ein Hochregallager kann man sich als ein 3 dimensionales Gitter vorstellen, in dessen Gitterzellen Warenpaletten eingelagert werden können. Typischerweise besitzt so ein Gitter viele 1000 Zellen. Der Zugriff auf die Gitterzellen erfolgt automatisch mit Hilfe von Regalbediengeräten welche sich in Gassen ins Innere des Lagers bewegen können und mit Hilfe von Förderbändern. Es soll zunächst ein Softwaretool erstellt werden, um solche Hochregallager zusammen mit den dazugehörigen Komponenten und Zugriffsmechanismen zu beschreiben und zu visualisieren. Im nächsten Schritt soll ein Simulator erstellt werden, mit dem ein Hochregallager (virtuell) betrieben werden kann, indem zufällige Sequenzen von Lagerzugriffen verarbeitet wird. Zweck des Simulators ist dabei das Austesten von verschiedenen Lagerverwaltungs-Strategien.

Gruppengrösse: 2-3

Anzuwendende Technologien: JAVA

Links: Hochregallager in Wikipedia

Betreuer: Peter Schwab
